# 테트리스

HTML, CSS, JavaScript만으로 만든 브라우저 테트리스 게임입니다.  
입문자가 웹 프론트엔드 기초를 배우기 위한 **교육용 프로젝트**로 제작되었습니다.

## 프로젝트 소개

| 항목 | 내용 |
|---|---|
| 기술 스택 | HTML, CSS, JavaScript (Vanilla) |
| 빌드 도구 | 없음 |
| 외부 라이브러리 | 없음 |
| 보드 크기 | 10열 × 20행 |
| 블록 종류 | I, O, T, S, Z, J, L (7종) |

별도 설치 없이 브라우저에서 바로 실행할 수 있으며, GitHub Pages에 정적 사이트로 배포할 수 있습니다.

## 실행 방법

### 로컬

1. 이 프로젝트 폴더를 연다.
2. `index.html` 파일을 더블클릭하거나 브라우저로 드래그한다.
3. 또는 브라우저 주소창에 파일 경로를 입력한다.

   ```
   file:///C:/DEV/tetris-cursor/index.html
   ```

### GitHub Pages

배포가 완료되면 아래 주소로 접속한다.

```
https://<GitHub사용자명>.github.io/<저장소명>/
```

> 저장소 루트에 `index.html`이 있으므로 `/src/` 경로 없이 루트 URL로 바로 접속할 수 있습니다.

## 파일 구조

```
tetris-cursor/
├── index.html   # 게임 화면 구조
├── style.css    # 스타일
├── script.js    # 게임 로직
└── README.md    # 프로젝트 안내
```

| 파일 | 역할 |
|---|---|
| `index.html` | 게임 보드, 점수, 버튼, 조작법 UI |
| `style.css` | 레이아웃, 보드 그리드, 버튼·패널 스타일 |
| `script.js` | 블록 생성, 낙하, 충돌, 라인 삭제, 점수, 게임 상태 |

## 구현 기능

### 게임 플레이

- 10×20 보드 렌더링 (CSS Grid)
- 7종 테트로미노 생성 및 표시
- 자동 낙하 (800ms 간격)
- 좌우 이동, 회전, soft drop, hard drop
- 바닥·고정 블록 충돌 판정 (`canMove`)
- 블록 고정 후 라인 삭제
- 줄 수에 따른 점수 증가
- 새 블록 스폰 불가 시 게임 오버

### 버튼

| 버튼 | 동작 |
|---|---|
| **시작** | 대기 상태에서 플레이 시작 (자동 낙하·키 입력 활성화) |
| **재시작** | 보드·점수·타이머 초기화 후 즉시 재플레이 |
| **다시 하기** | 게임 오버 후 표시되며, 재시작과 동일하게 동작 |

### 게임 상태

```
[대기] ──시작──▶ [플레이 중] ──스폰 불가──▶ [게임 오버]
                      ▲                           │
                      └──────── 재시작 / 다시 하기 ─┘
```

## 조작법

게임이 **진행 중**일 때만 키 입력이 적용됩니다.  
벽·고정 블록과 충돌하면 이동·회전은 취소됩니다.

| 키 | 동작 |
|---|---|
| `ArrowLeft` (←) | 왼쪽으로 한 칸 이동 |
| `ArrowRight` (→) | 오른쪽으로 한 칸 이동 |
| `ArrowDown` (↓) | 한 칸 빠르게 내리기 (soft drop) |
| `ArrowUp` (↑) | 블록 회전 (충돌 시 취소) |
| `Space` | 즉시 바닥까지 낙하 (hard drop) |

## 점수 규칙

블록이 고정된 뒤, **한 번에** 삭제된 줄 수에 따라 점수가 더해집니다.

| 한 번에 삭제한 줄 수 | 점수 |
|---|---|
| 1줄 | 100 |
| 2줄 | 300 |
| 3줄 | 500 |
| 4줄 | 800 |

## GitHub Pages 배포 방법

### 1. 저장소 생성 및 push

```bash
git init
git add index.html style.css script.js README.md
git commit -m "Add Tetris game"
git branch -M main
git remote add origin https://github.com/<USER>/<REPO>.git
git push -u origin main
```

### 2. Pages 설정

1. GitHub 저장소 → **Settings** → **Pages**
2. **Source**: `Deploy from a branch`
3. **Branch**: `main` / `/ (root)`
4. **Save** 클릭

### 3. 배포 확인

1~2분 후 아래 URL에서 게임이 실행되는지 확인합니다.

```
https://<USER>.github.io/<REPO>/
```

## 품질 점검 방법

배포 전 아래 항목을 순서대로 확인합니다.

| # | 확인 항목 | 기대 결과 |
|---|---|---|
| 1 | 페이지 로드 | 10×20 보드, 점수 0, 조작법 패널 표시 |
| 2 | 대기 중 키 입력 | 블록이 움직이지 않음 |
| 3 | **시작** 클릭 | 자동 낙하 시작, 상태 메시지 사라짐 |
| 4 | 좌우 이동 | `←` `→`로 이동, 벽에서 멈춤 |
| 5 | 회전 | `↑`로 회전, 충돌 시 취소 |
| 6 | soft / hard drop | `↓` 한 칸 낙하, `Space` 즉시 바닥까지 낙하 |
| 7 | 라인 클리어 | 가득 찬 줄 삭제, 점수 증가 |
| 8 | 게임 오버 | 보드 상단까지 차면 "게임 오버", 입력 중단 |
| 9 | **다시 하기** | 보드·점수 초기화, 재플레이 |
| 10 | 콘솔 | F12 → Console에 에러 없음 |
